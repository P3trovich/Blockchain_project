# Запуск и тестирование PharmaTrack

Пошаговая инструкция «с нуля до работающего демо». Если что-то идёт не так —
смотри раздел «Если что-то сломалось» в конце.

---

## 0. Что должно быть установлено

- **Node.js 18 или новее.** Проверка: `node --version`. Если нет —
  https://nodejs.org, ставь LTS-версию.
- **Браузер** — Chrome, Edge или Firefox. Yandex/Opera тоже работают, но
  с ними чаще ловят баги MetaMask.
- **MetaMask** — расширение для браузера: https://metamask.io.
  Ставится в обычное окно, не в сайдбар.
- **Git** (опционально, чтобы клонировать репу).

Python, Docker, Hyperledger и прочие тяжёлые штуки — **не нужны**.
Локальный блокчейн поднимает сам Hardhat в одном процессе Node.

---

## 1. Запустить три процесса

Нужно три открытых терминала (PowerShell или cmd). Все команды
выполняются из корня проекта (там, где `package.json`).

### Терминал 1 — установка зависимостей (только первый раз)

```powershell
npm install
```

Скачает Hardhat и всё, что нужно. Идёт ~1 минуту.

### Терминал 2 — локальная блокчейн-нода

```powershell
npx hardhat node
```

Должен показать список из 20 тестовых аккаунтов с приватными ключами и
балансом 10000 ETH у каждого. Этот терминал не закрывай — пока он работает,
работает блокчейн.

### Терминал 3 — деплой контракта

```powershell
npx hardhat run scripts/deploy.js --network localhost
```

Должен напечатать:

```
deploying from: 0xf39F...2266
PharmaTrack deployed at: 0x5FbDB...0aa3
ABI written to frontend/contract.json
granted LOGISTICS    -> 0x7099...79C8
granted DISTRIBUTOR  -> 0x3C44...93BC
granted PHARMACY     -> 0x90F7...b906
granted REGULATOR    -> 0x15d3...6A65
```

Если перезапускаешь ноду — **обязательно** перезапусти и деплой, иначе
адрес контракта на фронте будет указывать на ничто.

### Терминал 3 (тот же, после деплоя) — фронт

```powershell
node frontend/serve.js 5173
```

Покажет `frontend on http://127.0.0.1:5173`. Этот терминал тоже не закрывай.

Альтернатива, если у тебя есть Python:

```powershell
cd frontend
python -m http.server 5173
```

## 2. Настроить MetaMask

### 2.1. Создать кошелёк

При первом запуске MetaMask жми **«Создать новый кошелёк»**. Пароль —
любой (это локальная защита расширения, не сид-фразы). Сид-фразу из 12
слов записывать **не обязательно** — мы работаем с тестовой сетью, никаких
реальных денег там нет.

### 2.2. Добавить локальную сеть Hardhat

В новых версиях MetaMask сети добавляются с главного экрана:

1. Сверху слева кликни на индикатор сети (по умолчанию там «Ethereum
   Mainnet»).
2. В открывшемся списке внизу — кнопка **«+ Добавить пользовательскую
   сеть»** (или «Add custom network»).
3. Заполни:
   - Имя сети: `Hardhat Local`
   - URL RPC: `http://127.0.0.1:8545`
   - ID блокчейна: `31337`
   - Символ валюты: `ETH`
   - URL обозревателя блоков: оставь пустым.
4. Сохранить. MetaMask покажет жёлтые предупреждения про «GoChain Testnet»
   — это нормально, игнорируй и сохраняй.
5. Переключись на сеть `Hardhat Local`.

### 2.3. Импортировать тестовые аккаунты

Hardhat-нода даёт 20 готовых аккаунтов с фейковыми 10000 ETH у каждого.
Нам нужно импортировать первые 5 — по одному на каждую роль в системе.

Как импортировать в новых версиях MetaMask:

1. Открой MetaMask **на всю вкладку**: иконка ☰ (или иконка квадрата со
   стрелкой ↗) → «Развернуть в полный экран» / «Expand view».
2. В полноэкранном режиме клик по аватарке аккаунта в правом верхнем углу.
3. В открывшемся списке внизу — **«Импортировать аккаунт»** → тип
   «Закрытый ключ» → вставь ключ → «Импорт».

Ключи (это стандартные дефолтные ключи Hardhat — фейковые, безопасные,
их знает любой разработчик):

| Роль | Адрес | Приватный ключ |
|---|---|---|
| Admin / завод | `0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266` | `0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80` |
| LOGISTICS | `0x70997970C51812dc3A010C7d01b50e0d17dc79C8` | `0x59c6995e998f97a5a0044966f0945389dc9e86dae88c7a8412f4603b6b78690d` |
| DISTRIBUTOR | `0x3C44CdDdB6a900fa2b585dd299e03d12FA4293BC` | `0x5de4111afa1a4b94908f83103eb1f1706367c2e68ca870fc3fb9a804cdab365a` |
| PHARMACY | `0x90F79bf6EB2c4f870365E785982E1f101E93b906` | `0x7c852118294e51e653712a81e05800f419141751be58f605c371e15141b007a6` |
| REGULATOR | `0x15d34AAf54267DB7D7c367839AAf71A00a2C6A65` | `0x47e179ec197488593b187f80a00eb0da91f1b9d0b13f8733639f19c30a34926a` |

Назови импортированные аккаунты понятно: «Завод», «Логист»,
«Дистрибьютор», «Аптека», «Регулятор» — потом будет проще переключаться.

После импорта у каждого аккаунта на сети `Hardhat Local` должно быть
**10000 ETH**.

## 3. Сценарий тестирования

Открой http://127.0.0.1:5173 в том же браузере, где MetaMask.

### 3.1. Подключить кошелёк

1. Жми **«Подключить кошелёк»** (зелёная кнопка справа сверху).
2. MetaMask спросит, какие аккаунты подключить — выбери все пять,
   подтверди. Сейчас активным будет тот, что выбран в MetaMask — для
   первого шага это должен быть «Завод».
3. Под кнопкой появится адрес `0xf39F...2266`.

### 3.2. Проверка препарата (read-only)

1. Карточка **«Проверить препарат»** → серийник `SN-001` → «Проверить».
2. Ответ: `valid: false, status: "not_found"` — препарата ещё нет.

Это публичный запрос, не требует MetaMask. То, что делал бы пациент через
QR-сканер.

### 3.3. Регистрация препарата

1. Убедись, что в MetaMask активен аккаунт «Завод».
2. Карточка **«Зарегистрировать препарат»** — поля уже заполнены.
3. Жми «Зарегистрировать» → MetaMask откроет окно подтверждения
   транзакции → жми «Подтвердить».
4. Через 1–2 секунды под кнопкой появится `{ ok: true, gasUsed: ..., block: ... }`.
5. Снова проверь препарат через карточку «Проверить» → теперь
   `valid: true, status: "authentic", productName: "Парацетамол 500мг"`.

### 3.4. IoT-температура и автоматическая блокировка

1. Переключи MetaMask на аккаунт **«Логист»** (LOGISTICS).
2. Карточка **«Записать температуру (IoT)»**:
   - серийник: `SN-001`
   - температура: `50` (это 5.0°C — норма) → «Отправить» → подтвердить.
   - Ответ: `isBreach: false`.
3. Теперь нарушение: температура `200` (20.0°C — выше допустимых 8°C):
   - Отправить → подтвердить → `isBreach: true`.
4. Повтори ещё дважды (всего 3 нарушения).
5. **Контракт автоматически заблокирует препарат** на третьем нарушении
   (порог зашит в Solidity, `BREACH_THRESHOLD = 3`).
6. Проверь препарат через «Проверить» → `valid: false, status: "temperature breach threshold"`.

Это ключевой пункт демо: бизнес-правило живёт в смарт-контракте, его
нельзя обойти ни через UI, ни через прямой API-вызов.

### 3.5. Полный жизненный цикл (на незаблокированном препарате)

Зарегистрируй второй препарат с серийником `SN-002` (просто поменяй поле
и нажми «Зарегистрировать» от Завода).

1. **Завод** → карточка «Передачи и продажа», серийник `SN-002`,
   лицензия `DIST-LIC-1` → кнопка «→ дистрибьютору».
2. Переключи MetaMask на **Дистрибьютор** → лицензия `PHARM-LIC-9` →
   «→ в аптеку».
3. Переключи на **Аптека** → «→ продать пациенту».
4. Карточка «История препарата» → `SN-002` → увидишь полный путь:
   Register → ToDistributor → ToPharmacy → Sale.

### 3.6. Регуляторная блокировка

1. Зарегистрируй ещё препарат `SN-003`.
2. Переключи MetaMask на **Регулятор**.
3. В карточке «Передачи и продажа» серийник `SN-003` → красная кнопка
   «блокировать» → введи причину (например, «контрафакт») → подтверди.
4. Проверь препарат → статус: `контрафакт`, `valid: false`.

Это сценарий, когда препарат прошёл все проверки, но регулятор отозвал
партию post factum.

### 3.7. Запуск автотестов смарт-контракта

Отдельная демонстрация для защиты — что контракт покрыт юнит-тестами:

```powershell
npx hardhat test
```

Должно показать 8 зелёных тестов:

```
PharmaTrack
  ✔ регистрирует препарат и не даёт зарегистрировать дубль
  ✔ проверяет роли — посторонний не может регистрировать
  ✔ записывает температуру и помечает нарушения
  ✔ после трёх нарушений препарат блокируется автоматически
  ✔ полный путь: завод → дистрибьютор → аптека → продажа
  ✔ нельзя передать в аптеку, если была нарушена холодовая цепь
  ✔ регулятор может заблокировать препарат
  ✔ verifyDrug возвращает not_found для незнакомого серийника

8 passing
```

## 4. Если что-то сломалось

### Фронт показывает «MetaMask не найден»

- MetaMask стоит в другом браузере, чем открыт фронт. Открой
  http://127.0.0.1:5173 в браузере с расширением.
- Окно в режиме инкогнито — расширения там по умолчанию выключены.
- MetaMask стоит как сайдбар (Edge), а не как обычное расширение —
  закрепи иконку лисы рядом с адресной строкой.
- Обнови страницу через Ctrl+F5.

### MetaMask пишет «Internal JSON-RPC error» при подтверждении

- Hardhat-нода не запущена или была перезапущена. Проверь, что
  `npx hardhat node` работает.
- После рестарта ноды обязательно перезаливай контракт (`npx hardhat run scripts/deploy.js --network localhost`).
- В MetaMask: «Настройки» → «Расширенные» → «Очистить данные активности
  вкладки» (Reset account / Clear activity tab data) — это сбрасывает
  локальный nonce.

### Транзакция «вечно ждёт подтверждения» в MetaMask

То же самое — рассинхрон nonce между MetaMask и hardhat-нодой. Лечится
сбросом activity, см. выше.

### Фронт пишет «Сначала задеплойте контракт»

Файл `frontend/contract.json` не создан. Запусти deploy-скрипт ещё раз.

### `forbidden` при нажатии кнопки

Аккаунт в MetaMask не имеет нужной роли для этой операции. Проверь, что:
- регистрацию делает Завод,
- температуру пишет Логист,
- передачу в аптеку делает Дистрибьютор и т.д.

### Порт 5173 или 8545 занят

Поменяй: для фронта — `node frontend/serve.js 5174`. Для hardhat-ноды
— это сложнее, проще убить процесс, держащий 8545.

### Хочется начать с чистого листа

Останови оба процесса (Ctrl+C в терминалах), удали `frontend/contract.json`,
запусти заново ноду и деплой. В MetaMask — «Очистить данные активности».

## 5. Что показывать на защите

Минимальный сценарий, который проходится за 4–5 минут:

1. Открыть `contracts/PharmaTrack.sol` — пройтись по структуре, показать
   роли и проверки.
2. `npx hardhat test` — 8 зелёных тестов.
3. Запустить ноду и деплой, открыть фронт.
4. Зарегистрировать препарат от завода → проверить.
5. Записать 3 нарушения температуры от логиста → препарат
   автоматически блокируется → проверить.
6. Открыть `frontend/contract.json` или devtools → показать, что фронт
   читает прямо из контракта, никакой централизованной БД нет.
